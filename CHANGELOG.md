# 変更履歴

このファイルには、利用者に影響する変更を記録する。細かな内部変更はGitのコミット履歴で管理する。

## 未リリース

### 変更

- 「実行後」は循環切り替えだけに表示し、ボタンは成功後に閉じる動作へ固定。
- トラブルシューティングのフォルダー操作を「データフォルダーを開く」へ集約。
- 起動時の環境情報を通常ログへ記録し、モニター構成、DPI、作業領域等が変化した場合だけ再記録するよう変更。

### 修正

- スタートボタンを連続クリックした際、退出中の再表示を取りこぼしてランチャーが起動しなくなる問題を修正。
- 複数モニターで別のSearchHostウィンドウを選び、スタートとランチャーが異なるディスプレイへ表示される問題を修正。
- ログ削除後、アプリを再起動するまで環境情報がログへ再記録されない問題を修正。

## 0.5.1-dev - 2026-07-21

### 追加

- トラブルシューティング画面と診断情報にアプリバージョンを表示。

### 修正

- 診断情報でWindowsの表示バージョンと完全なOSビルド番号を表示。
- UI Automationイベント登録が長時間停止する環境でも、Windowsキーから起動できるよう監視スレッドを分離。
- マルチモニター環境でSearchHostの全面ウィンドウをスタートメニューと誤認し、表示を中止する問題を修正。
- 設定画面を開いた直後、「実行後」が空欄になる問題を修正。

### 変更

- ボタンの実行種類「コマンドを実行」を「コマンド／batを実行」へ変更。

## 0.5.0-dev

- Phase 5の対応環境検証と初回配布前テストを実施中。
