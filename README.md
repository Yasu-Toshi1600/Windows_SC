# Windows_SC

Windows 11のスタートメニューと一緒に表示できる、常駐型の個人向けランチャーです。
アプリ、ファイル、フォルダー、URL、コマンド、音声出力先、マスター音量を
一つの画面から操作できます。

現在のバージョンは`0.6.1-dev`です。開発・配布前確認中のため、正式版では
ありません。

## 対応環境

- Windows 11 25H2以降（OSビルド26200以降）
- x64
- 下端タスクバー
- 720p以上でランチャーを配置できる作業領域

Windows 10、Windows 11 24H2以前、x86／ARM64、下端以外のタスクバーは
現在のサポート対象外です。起動時に対応環境を検査し、対象外の場合は理由を
表示して終了します。

## 基本操作

アプリを起動すると通知領域へ常駐します。ランチャーは次の方法で表示できます。

- Windowsキー単体
- タスクバーのスタートボタン
- `Ctrl+Alt+Space`
- 通知領域アイコンの「ランチャーを表示」

通知領域アイコンから、ランチャー表示、設定、アプリ終了を選択できます。

## 登録できる項目

- ボタン
  - アプリ、ファイル、フォルダー、URLを開く
  - コマンド／batを実行する
- 循環切り替え
  - 音声出力デバイスを登録順に切り替える
  - 複数のコマンドを登録順に実行する
- 音量スライダー
  - 現在の既定音声出力のマスター音量を変更する

起動対象欄のフォルダーアイコンから、ファイルまたはフォルダーを選択できます。
PATHで解決できる`notepad.exe`等、URL、Windows URIは手入力も可能です。

## 設定

設定は自動保存されません。設定画面下部の「保存」を押した時点で、入力検証、
設定ファイル保存、自動起動の反映、ランチャー表示の更新を行います。

主な設定:

- スマートフォン連携パネル用の表示位置調整
- 標準／コンパクト表示
- Windowsログオン時の自動起動
- ランチャー項目の追加、編集、削除、並べ替え
- 通常ログと詳細診断

## データ保存場所

設定とログは次へ保存します。

```text
%LOCALAPPDATA%\Windows_SC\
├─ Settings\
└─ Logs\
```

設定画面の「データフォルダーを開く」から確認できます。アプリが設定やログを
外部へ自動送信することはありません。詳細ログにはコマンドや音声デバイス名等が
含まれる可能性があるため、共有前に内容を確認してください。

## 開発用ビルド

```powershell
dotnet build Windows_SC.slnx -c Debug -p:Platform=x64
dotnet build Windows_SC.slnx -c Release -p:Platform=x64
```

配布物はx64自己完結版を使用する方針です。自己完結版の利用先にVisual Studioや
.NET SDKは不要ですが、現在は配布方式と自動起動の保証範囲を最終決定中です。
