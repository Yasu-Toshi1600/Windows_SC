# ログ・詳細診断・プライバシー仕様

作成日: 2026-07-21
対象: Windows_SC Phase 5

## 1. 目的

通常利用で必要な障害情報を残しながら、UI Automation候補、座標、デバイス識別子、実行対象などの調査情報を通常ログから分離する。ログはローカル保存だけとし、Windows_SCからネットワークへ送信しない。

## 2. 保存場所

```text
%LOCALAPPDATA%\Windows_SC\Logs\window-diagnostics.log
%LOCALAPPDATA%\Windows_SC\Logs\window-diagnostics.previous.log
%LOCALAPPDATA%\Windows_SC\Logs\window-diagnostics.detail.log
%LOCALAPPDATA%\Windows_SC\Logs\window-diagnostics.detail.previous.log
```

- UTF-8、BOMなし、各行にローカル日時とミリ秒を付ける。
- 通常ログと詳細ログはそれぞれ起動時に2MB以上ならpreviousへ移動する。
- previousは各種類1世代だけ保持する。
- 設定画面の削除操作では4ファイルを確認後に削除する。

## 3. 通常ログ

通常ログは常に有効とし、次を記録する。

- アプリの起動、二重起動拒否、終了
- 設定の作成、読込、保存、破損復旧
- スタート検出結果と監視状態
- ランチャーの状態遷移、表示・退出理由
- 表示とモーションの主要時間
- アクション、音声、トレイ操作の成否
- 例外種別、HRESULT、Win32エラー番号

通常ログには次を直接記録しない。

- UI Automation要素名、AutomationId、候補一覧
- PID以外の詳細なウィンドウ情報、HWND、矩形
- 音声デバイスIDとデバイス名
- アクションの実行対象
- Windowsキーの低レベルなdown／up分類

## 4. 詳細診断ログ

設定画面の「トラブルシューティング」で有効にする。保存時点から24時間だけ有効で、期限後は自動的に詳細記録を停止する。

詳細ログには有効期間中の通常ログも複製する。そのため、調査時は詳細ログだけで全体の時系列を追える。通常ログへは詳細項目を逆流させない。

追加で記録する項目:

- UI Automationワーカーとフォーカス監視の開始・終了
- フォーカス要素、スタート候補、AutomationId、矩形
- ランチャーの配置座標、サイズ、フォーカス状態
- 描画準備、Loaded、表示状態
- Windowsキーの単体／ショートカット分類
- アクション実行対象と詳細エラーメッセージ
- 音声切り替え先のデバイス名とID

## 5. パス匿名化

通常ログと詳細ログの両方で、ログキューへ入れる前に匿名化する。

既知のルートは次のトークンへ置き換える。

```text
アプリ配置フォルダー  → %APPDIR%
ローカルAppData       → %LOCALAPPDATA%
ローミングAppData     → %APPDATA%
一時フォルダー        → %TEMP%
ユーザープロファイル  → %USERPROFILE%
```

トークンより後ろのパス要素は `<redacted>` とする。別ユーザーの `C:\Users\名前` が例外メッセージに含まれた場合も、ユーザー名を `<user>` へ置き換える。改行とNULL文字は空白へ置き換え、ログ行の注入を防止する。

匿名化は共有前確認の代わりにはならない。UI Automation要素名、音声デバイス名、ファイル名以外の例外情報にも個人情報が含まれる可能性があるため、ログを外部へ渡す前に利用者が内容を確認する。

## 6. 設定画面

別のトップレベルウィンドウは作らず、既存設定画面の下部からContentDialogを開く。

- 詳細診断ログのオン／オフ
- 現在状態と有効期限
- 通常・詳細ログの保存場所
- ログフォルダーを開く
- OS、アーキテクチャ、.NET、アプリ版、モニター数、解像度、配置、作業領域、DPI、拡大率、取得可能なリフレッシュレートの環境情報をコピー
- 確認後にログを削除

詳細診断ログの切り替えは、ダイアログ内の「詳細診断ログ設定を適用」で確定する。項目編集に無効値や未保存変更があっても、診断機能だけを有効化できる。設定画面下部の「保存」を使った場合も現在の診断設定を保持する。

## 7. 実装上のルール

- 利用者向けの状態、成否、主要時間は `DiagnosticLogger.Write` を使う。
- 個人情報またはOS内部情報を含み得る調査値は `DiagnosticLogger.WriteDetailed` を使う。
- 呼び出し側だけの文字列置換へ依存せず、最終的な匿名化はlogger境界で必ず行う。
- 詳細ログが無効なとき、詳細メッセージはキューにもファイルにも入れない。
- ログ書込みはバックグラウンドライターで行い、UIスレッドから同期ファイルI/Oを行わない。
- 新しいログ項目を追加した場合、通常／詳細のどちらに属するかと、共有時の情報漏えいリスクをレビューする。

## 8. 確認項目

1. 無効時に詳細ログへ新しい行が追加されない。
2. 有効化して保存すると通常イベントが両ファイルへ、詳細イベントが詳細ファイルだけへ入る。
3. 24時間経過後に詳細記録が停止する。
4. 再起動後も期限内だけ有効になる。
5. ログにWindowsユーザー名とユーザープロファイルの実パスが残らない。
6. ログ削除のキャンセルでは何も消えず、確定時に4ファイルが削除される。
7. ログフォルダーと環境情報に不要な個人情報が含まれない。
