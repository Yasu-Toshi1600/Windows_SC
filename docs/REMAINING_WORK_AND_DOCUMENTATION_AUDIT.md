# 現行修正課題・ドキュメント整理表

更新日: 2026-07-22  
対象: `0.5.6-dev` の現行ワークツリー

## 1. この文書の役割

初回個人配布までに残っている実装修正、追加検証、ドキュメント整理を一か所で管理する。
詳細な仕様は各専門文書を正とし、この文書には「何が未完了か」「どの文書を残すか」を記載する。

対応環境は Windows 11 25H2以降、日本語環境、x64、下端タスクバーを基本とする。低解像度向けの可変レイアウト対応は行わず、最低720pかつランチャーを収められる作業領域を前提とする。対応範囲外でも例外終了させないための簡単な領域確認は、必要になった場合だけ追加する。

## 2. 修正・検証が必要な項目

### 優先度A: 初回配布前に解消する

| ID | 項目 | 現状と問題 | 完了条件 |
|---|---|---|---|
| R-01 | スタート連動の最終状態統合 | 実装・実機確認済み。Explorerの`StartButton`矩形と同じモニターのWin32候補だけを採用し、未操作のスタート連動中は最後のVisible／Hidden要求へ収束する。Hide完了時にも最新Snapshotを再確認し、連続クリック後に起動しなくなる問題は現時点で再発していない。 | 修正済みとして回帰試験へ移し、スタート関連を変更した場合だけ通常操作、連続クリック、退出途中の再表示を複数モニターで再確認する。 |
| R-02 | 設定検証の強化 | 実装済み。全列挙値、null、種類ごとのAction／CycleAction／VolumeSliderと必須値を検証し、空のボタン対象は保存前に日本語で指摘する。新規既定値も検証可能な実行項目へ変更した。 | 不正JSONを用いた破損復旧と、設定画面を開いても例外にならないことを実機確認する。 |
| R-03 | 未処理例外の診断 | 実装済み。UIスレッド、AppDomain、未監視Task、終了保存失敗を通常ログへ記録し、致命的例外を処理済みにはしない。 | 安全な試験用例外でログ項目を確認し、異常終了を隠して継続しない。 |
| R-04 | 設定保存と自動起動の整合性 | 実装済み。JSON保存を先に行い、自動起動反映失敗時は設定ファイルとレジストリを保存前へ戻し、実際の状態を画面とログへ表示する。起動時の同期失敗はログへ残して常駐を継続する。 | 通常保存と、書込みまたはレジストリ反映を失敗させた試験で部分反映が残らないことを確認する。 |
| R-05 | 配布物の再作成 | 連続スタートクリック、モニター選択、ボタンの実行後表示修正は、既存の配布ZIPより新しいワークツリーにある。 | 修正をコミット後、バージョンを判別できるx64自己完結版を再作成し、別PCでファイル版とログ版が一致することを確認する。 |
| R-13 | 手動表示時の前面化と画面外クリック | `Ctrl+Alt+Space`直後の画面外クリック問題は、50ms間隔、最大5回の前面化再要求を実装して実機確認済み。2026-07-22にVM上の`0.5.3-dev`を継続動作中、一時的に最前面へ表示されない別事象が1回発生したが、プロセス再起動後に解消し、同じ環境では再現していない。発生時は詳細ログが無効で、通常ログにも`activation-result=failed`がなかった。 | `Ctrl+Alt+Space`の既知バグは修正済みとして回帰試験へ移す。VMの最前面事象だけを配布停止を伴わない経過観察とし、再発時はプロセスを終了する前に操作経路、直前の前面アプリ、発生時刻、VMの表示状態を記録する。 |
| R-14 | 仮想デスクトップ切り替え後の表示 | 実装・実機確認済み。表示前に`IVirtualDesktopManager`で現在の仮想デスクトップ所属を確認し、別デスクトップなら現在操作中のデスクトップへ移動する。移動時は古いVisible／Exiting状態をHiddenへリセットし、最初の入力を非表示ではなく表示として処理する。 | 修正済みとして回帰試験へ移し、表示処理を変更した場合はWindowsキー、スタートボタン、`Ctrl+Alt+Space`、トレイの全経路でデスクトップ1→2→1を再確認する。 |

### 優先度B: 配布前に仕様を決め、必要なら修正する

| ID | 項目 | 現状と問題 | 推奨方針 |
|---|---|---|---|
| R-06 | コマンド循環の再実行設定 | 仕様決定・実装済み。`cmd.exe`の起動失敗はまれで、コマンド自身の終了コードを追跡しない現行方式では誤解を招くため、「失敗時は同じ操作を再実行する」をUI、モデル、保存データから削除した。旧JSONの同名プロパティは読み込み時に無視される。 | 修正済みとして回帰試験へ移し、コマンド循環が起動要求の成否にかかわらず操作ごとに次へ進むことを確認する。終了コード判定は将来機能とする。 |
| R-07 | 詳細ログの個人情報 | パスは匿名化するが、UI Automation要素名にはDiscord等のアカウント名や他アプリの表示内容が含まれ得る。詳細ログは完全匿名ではない。 | スタート判定に不要な要素名を記録しないか伏せ字にする。少なくとも画面と文書で「共有前確認が必要」と明示する。 |
| R-08 | 詳細ログ期限 | トラブルシューティングの適用だけでなく、一般設定を保存するたび有効期限が現在時刻から24時間へ延長される。文書上の「診断設定を保持する」と一致しない。 | 一般設定の保存では既存期限を維持し、診断の専用適用時だけ24時間を再設定する。 |
| R-09 | 設定画面の音声デバイス更新 | 設定画面を開いた時点のキャッシュだけを一覧へ入れる。開いたまま接続・切断しても追加候補と利用可否が更新されない。 | 音声状態変更イベントで一覧を更新するか、明示的な再取得ボタンを設ける。 |
| R-10 | 起動時の同期処理 | 設定読込と一部の音声初期化を同期的に待つため、ストレージ、セキュリティ製品、音声スタックが遅い環境では起動が固まったように見える可能性がある。 | 入力・トレイ・終了を先に利用可能にし、音声列挙等はバックグラウンドで完了させる。起動時間を通常ログで区間計測する。 |
| R-11 | 対応環境の検査 | 文書ではWindows 11 25H2以降、x64、下端タスクバー限定だが、起動時の厳密な判定は未実装で、プロジェクトにはx86／ARM64定義も残る。 | 配布対象はx64だけに固定する。OS／タスクバーを強制終了条件にするか、警告付き非対応とするかを決める。 |
| R-12 | ログローテーション | 2MB判定はロガー初期化時だけで、長時間起動中に2MBを超えても次回起動までローテーションしない。詳細ログの連続記録ではファイルが大きくなり得る。 | 初回個人配布では起動時ローテーションとして明記する。長時間試験で問題になる場合だけ実行中ローテーションを追加する。 |

### 優先度C: 初回配布後でもよい

- 二重起動時に既存プロセスを前面表示する機能。現状は二重常駐を防止して終了するだけである。
- 代替グローバルショートカットを設定画面から変更する機能。
- テーマ、色、アイコン等の外観カスタマイズ。
- コマンドの終了待機、終了コード、タイムアウト、標準出力取得。
- 低解像度向けの可変高さ・専用レイアウト。初回版の対応対象には含めない。

## 4. 今回確認した古い記述

明確に現仕様と食い違う記述は2026-07-22に元文書も修正した。構成の分割や実装変更を伴う項目は残件として管理する。

### 旧`INITIAL_DESIGN.md`

- ボタンにも「実行後」を選択できる記述があった。現行仕様の、ボタンは成功後に閉じ、循環切り替えだけ選択可能という記述へ修正した。
- 「環境変数を扱わない」とあったが、現実装に合わせ、起動対象と作業フォルダーで環境変数を展開すると修正した。
- クラス構成は実装前の構想名が多く、現在のファイル構成を表していない。
- Phase 4の進捗にボタンの実行後動作を含めていたため修正した。
- テスト項目を本文内にも持っており、Phase 5～6手順書と重複している。

旧初期設計書は`archive`へ移し、現行概要と専門文書への索引を持つ`DESIGN.md`へ置き換えた。

### `MOTION_REDESIGN_PROCEDURE.md`

- Win32を先に確認する記述を、フォーカス中のUI Automation矩形を優先し、取れない場合だけWin32へフォールバックする現行順序へ修正した。
- 退出中の反転を新しいWindowsキーだけに限定する記述を、スタート再クリックによる新しいSnapshotも受け入れる内容へ修正した。
- `PublishReadyToRun=false`やARM64ビルドの記録を当時の履歴と明示し、現在はx64、ReadyToRun有効、トリミング無効を正とした。

現在有効な内容と`MOTION_BASELINE.md`の計測値は`MOTION_SPECIFICATION.md`へ統合し、2つの元資料は`archive`へ移した。

### `PHASE5_6_TEST_PROCEDURE.md`

- 低解像度対応を正式な合格条件から外し、最低720pを前提とする内容へ修正した。
- SET-12を、ボタンは成功後に閉じ、循環切り替えだけ実行後動作を選ぶ内容へ修正した。
- SEC-02を、シェルとして実行するコマンド欄と、アプリ／ファイル起動欄のメタ文字規則を分ける内容へ修正した。
- 2MB到達時のローテーションは実行中ではなく次回起動時であると明記した。
- 日常確認と正式リリース判定が同じ文書にあり重かったため、日常用の`TEST_CHECKLIST.md`を新設し、詳細版は必要時だけ使う位置付けへ変更した。

### `START_MENU_INTEGRATION_MAINTENANCE.md`

- コンポーネント表の旧い「Win32優先スキャン」を、UI Automation優先の現行方式へ修正した。
- UI Automationで矩形を得られない最初のクリックにおける、複数`SearchHost`のモニター選択リスクを既知の制約へ追加した。

### `LOGGING_AND_PRIVACY.md`

- 一般設定保存時に詳細ログ期限を維持する設計と、24時間へ延長する現在の実装が一致しないため、設計上の正と実装残件を明記した。
- UI Automation要素名に他アプリのアカウント名等が含まれ得る点を、パス匿名化とは別の制約として強調した。
- 起動中のローテーションは未実装である。未処理例外ログは優先度A対応として実装済みで、試験用例外による確認を残す。

## 6. 整理の実施順

1. R-01～R-05を修正・検証し、現行配布物との差をなくす。
2. ログ仕様に残る実装差を解消する。
3. READMEを作成し、利用者がコード内部の設計書を読まなくても導入できるようにする。
4. x64自己完結版を再作成し、別PCで簡易チェックリストを行う。
